# 任务：给「哥布林堵城」做视觉翻新

你面对的是一个功能完整、但**故意零美术**的单文件 HTML 小游戏。你的任务是把它翻新成有质感、有反馈、手机可玩的成品。**只翻新表现层，游戏逻辑一个字符都不许动。**

**世界观与命名完全自由。**「哥布林」「城堡」「银块」「金块」只是下文描述机制用的工作命名——你可以整体重塑主题（换敌人、换材质比喻、换标题、换文案），只要机制语义不变、且信息通道保持可分辨（见铁律 5）。当然，直接沿用哥布林攻城的主题也可以。

## 游戏规则（供理解机制，命名可换、语义不许改）

- 标准俄罗斯方块盘面（10×20），方块从传送带供应。这是一场攻城战：盘面底部是**哥布林城堡（3 血）**，你的目标是砸空它的血破城通关；你自己有**城墙 3 血**，掉完判负。
- 哥布林层每 25 秒从盘面底部上顶一层，既把你往上挤、也是城堡的盾；**没有空间上顶时改为直击你的城墙 1 血**。哥布林层不可用普通消行清除。
- 砸层效果在场上已无哥布林层时，直接对城堡造成 1 点伤害。
- 传送带 8 槽左进右出、逐格前进，到头排队等待；方块**不会自动下落**，由玩家从传送带上挑选出战，场上同时只有一个下落块。
- 传送带交互：**点击方块 = 出战开始下落**；方块不可丢弃，只能出战或用于合成。
- 相邻 2 个同形状普通方块可合成**银块**，3 个合成**金块**，2 个银块也可合成金块。
- **银块落地**：四格作为银格留在墙里；含银格的行被消除时哥布林计时回拨 15 秒；银块落地直接消行则额外砸 1 层。
- **金块落地**：四格作为金格留在墙里；含金格的行被消除时砸 1 层；金块落地直接消行则合计砸 2 层。

## 铁律

1. **封条区**：`index.html` 中 `<script id="game-core">` 整块（含注释）逐字符保持原样。不得修改、不得在其他位置重新实现或覆盖其中的函数与状态。
2. **自由区**：`<script id="game-ui">` 可整体重写，CSS 与 DOM 结构随意改造。新增交互只能通过调用封条区导出的函数或 `window.GameAPI` 驱动游戏。
3. **单文件、零外部依赖**：不引入 CDN、网络字体、外链图片、第三方库。素材用 CSS、inline SVG 或 data URI。
4. **验收封条**：打开 `index.html?autoplay=1&seed=AUDIT05`，页面输出的 SEAL-REPORT 必须与 `baseline.md` 逐字一致。交卷前先自测。
5. 页面内的规则说明须与你选定的主题一致，并把机制讲清楚；机制语义不得改变。**信息通道不许合并**：两种合成材质必须一眼可分，威胁行必须读得出压迫，计时逼近必须可感知——皮肤随便换，玩家的判断依据一个都不能丢。

## 期望的表现方向（自由发挥）

以下时刻需要有存在感的反馈（按机制说，主题措辞随你的世界观走）：**威胁层上顶（压迫感）、计时告急、我方被直击掉血、时间材质行消除回拨计时、伤害材质入墙与引爆砸层、敌方核心受击、通关、合成、消行、硬降落地、传送带移动、挑块出战、战败**。两种材质要有质感差异，威胁要有敌意，敌方核心受击和通关是全场最大的两个演出时刻。手机 390×844 竖屏要能完整游玩（交互保留），桌面键盘操作保留。追求 60fps。
